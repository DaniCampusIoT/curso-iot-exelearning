Fase 6. Análisis, reflexión y entrega

📋 Fase 6 · Evaluación

Rúbrica de evaluación del proyecto

Esta rúbrica evalúa el dominio técnico, la comprensión del sistema IoT, la calidad de la presentación y la reflexión crítica sobre el proceso. Cada criterio refleja directamente las competencias trabajadas a lo largo de las seis fases del proyecto.

Cómo funciona esta rúbrica: cada criterio se puntúa del 1 al 4. La nota final se obtiene sumando los puntos obtenidos en cada criterio multiplicados por su peso porcentual. La máxima puntuación posible es 10 puntos.

Criterio Peso 4 — Excelente 3 — Bien 2 — En proceso 1 — Necesita mejorar
🔌 Montaje y hardware 25% Cableado I2C correcto (SDA D2, SCL D1, VCC 3.3V, GND), circuito limpio, sensor detectado en dirección 0x76 o 0x77, sistema 100% funcional. Circuito funcional con algún cable suelto o etiqueta incorrecta. El sensor responde con ayuda puntual del profesor. El circuito necesita correcciones frecuentes. El sensor no siempre se detecta o hay confusión entre pines. El circuito no funciona o el sensor no es detectado. Se requiere reconstruir el montaje desde el principio.
💻 Código y programación 20% El programa compila y sube sin errores. WiFi, MQTT y BMP280 funcionan. Entiende y explica wifiConnect(), sendToBroker() y loop(). El programa funciona. Puede localizar las funciones clave en el código pero necesita apoyo para explicar alguna parte. El programa funciona con modificaciones del profesor. No comprende bien la estructura del código. El programa no compila o no se conecta a la red. No identifica las partes del código.
📡 Comunicación IoT 20% El ESP8266 publica correctamente en el topic MQTT con el JSON bien estructurado. Entiende el rol del broker, la diferencia entre publish y subscribe, y la función de LWT. Los datos llegan al broker con la estructura JSON esperada. Comprende MQTT y los topics pero no domina LWT ni los detalles del JSON. Los datos llegan con errores de formato o topic incorrecto. Comprende MQTT de forma superficial. Los datos no llegan al broker o la estructura JSON es incorrecta. No comprende el protocolo MQTT.
📊 Dashboard Node-RED 25% Flujo completo y operativo: mqtt-in → json → function → ui_gauge / ui_text. Dashboard organizado con Tab, Group y widgets para temperatura, presión y datos del ESP. Explica cada nodo. Dashboard funcional con al menos temperatura visible. El flujo funciona pero algún widget está mal configurado o falta alguna variable. El flujo recibe datos pero los widgets no muestran los valores correctamente o el dashboard está incompleto. El flujo no recibe datos o el dashboard no tiene ningún widget funcional.
📝 Documentación y reflexión 10% Documenta el proceso, justifica decisiones técnicas, reflexiona sobre privacidad de los datos y uso responsable de la IA, si se ha utilizado. Documentación suficiente y reflexión correcta. Menciona la privacidad y el uso de herramientas de IA pero sin profundidad. Documentación básica e incompleta. Reflexión superficial o ausente. Sin documentación ni reflexión.

🧮 Cálculo de la nota final

Nota = (punt_hardware × 0.25 + punt_codigo × 0.20 + punt_mqtt × 0.20 + punt_dashboard × 0.25 + punt_doc × 0.10) × 2.5
Ejemplo: 4 + 3 + 4 + 3 + 4 → (1.0 + 0.6 + 0.8 + 0.75 + 0.4) × 2.5 = 8.875 → nota 8,9
