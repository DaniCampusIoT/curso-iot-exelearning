Fase 3. IoT, WiFi, MQTT y JSON

🌐 Fase 3 · 2 sesiones

IoT, WiFi, MQTT y JSON

En esta fase conocerás la arquitectura general del proyecto y comprenderás cómo un sensor físico puede enviar información a través de la red para ser visualizada posteriormente en un panel de datos. El objetivo no es solo programar, sino entender cómo viajan los datos desde el dispositivo hasta la interfaz final.

IoT WiFi MQTT Broker JSON Payload

🔗Arquitectura general del sistema

El proyecto sigue una estructura típica de Internet de las Cosas: un sensor mide una variable física, el microcontrolador la procesa, la red permite transmitirla, un servidor intermedio organiza los mensajes y una herramienta final los visualiza. Esta lógica ya aparece planteada en la secuencia didáctica de tu REA y servirá como base para las fases prácticas siguientes. 

Sensor BMP280 mide temperatura y presión.
Microcontrolador El ESP8266 procesa los datos.
WiFi Conecta el dispositivo a la red local.
Broker MQTT Recibe y distribuye los mensajes.
Dashboard Node-RED muestra los datos en tiempo real.

📡¿Qué es IoT?

El IoTInternet of Things o Internet de las Cosas. Se refiere a objetos físicos conectados a la red que pueden medir, intercambiar y procesar datos. permite que dispositivos cotidianos recojan información del entorno y la compartan automáticamente a través de Internet o de una red local. En este proyecto, la estación meteorológica actúa como un pequeño sistema IoT porque mide variables físicas y las envía a otro entorno para su visualización. 

Esta fase te ayuda a entender que el proyecto no se limita a leer un sensor, sino que forma parte de una cadena completa de adquisición, transmisión y uso de datos. 

📶WiFi y TCP/IP

El ESP8266 incorpora conectividad WiFiTecnología de red inalámbrica que permite conectar el microcontrolador a una red local sin usar cables., lo que permite que el sistema se comunique con otros equipos de la red. Una vez conectado, utiliza la lógica general de TCP/IPConjunto de protocolos que permite el envío de datos entre dispositivos conectados a una red. IP gestiona el direccionamiento y TCP garantiza la entrega correcta de los datos. para intercambiar información. 

En las fases siguientes esta conexión será fundamental para enviar las lecturas del sensor a un broker MQTT y después representarlas en Node-RED.

¿Qué es MQTT?

MQTTProtocolo de mensajería muy ligero, diseñado para dispositivos con pocos recursos y para redes donde no siempre hay mucha estabilidad o ancho de banda. es un protocolo de comunicación muy usado en IoT porque resulta simple, eficiente y ligero. En lugar de que un dispositivo pregunte continuamente por los datos, se usa un modelo de publicación y suscripción: un dispositivo publica y otro recibe solo si está suscrito al canal adecuado. 

Ese canal se llama tópicoRuta o nombre jerárquico que organiza los mensajes en MQTT. Permite separar y clasificar la información transmitida. y el servidor intermedio se llama brokerServidor que recibe los mensajes MQTT enviados por los publicadores y los distribuye a los clientes suscritos a cada tópico.. Así, el ESP8266 puede actuar como publicador y Node-RED como suscriptor. 

🧩¿Qué es JSON?

JSONFormato de texto ligero para intercambiar datos estructurados. Organiza la información en pares clave-valor y puede incluir objetos dentro de otros objetos. es un formato de texto muy cómodo para representar datos estructurados. Permite agrupar la información en claves y valores, lo que resulta muy útil para enviar varias medidas del sensor en un único mensaje. 

En este proyecto, el contenido del mensaje MQTT o payloadContenido útil del mensaje enviado por MQTT. En este caso contiene los datos del sensor en formato JSON. incluye la información del ESP8266 y los valores leídos del BMP280. Esta estructura hará más fácil extraer datos concretos cuando trabajes después con Node-RED. 

Idea clave: en esta fase no se trata todavía de enviar datos de verdad, sino de comprender la lógica de comunicación que hará posible esa transmisión en la fase práctica siguiente. Primero entiendes la arquitectura; después la implementas. 
Fragmento del proyecto · Construcción del JSON antes de publicarlo
// Construcción del documento JSON
DynamicJsonDocument doc(1024);

// Objeto con información del ESP8266
JsonObject esp = doc.createNestedObject("esp");
esp["ip"] = WiFi.localIP().toString();
esp["rssi"] = WiFi.RSSI();
esp["timestamp"] = millis();

// Objeto con datos del sensor
JsonObject values = doc.createNestedObject("sensor");
values["name"] = "BMP280";
values["temperatura_c"] = tempC;
values["presion_hpa"] = pressHP;
values["altitud_m"] = altM;

// Serialización del JSON para enviarlo por MQTT
char payload[1024];
serializeJson(doc, payload, sizeof(payload));
client.publish(send_topic, payload);
