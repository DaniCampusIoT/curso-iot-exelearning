Fase 2. Lectura local del sensor

⚡ Fase 2 · 2 sesiones

Lectura local del sensor

En esta fase cargarás tu primer sketch en el Wemos D1 mini, leerás los datos del sensor BMP280 en tiempo real y aprenderás a usar el monitor serie y el escáner I2C para depurar el circuito.

🛠 Arduino IDE 🔗 I2C 🌡 BMP280 🔍 I2C Scanner 📟 Monitor serie

🛠Software necesario

1
Arduino IDE 2.x
Descarga e instala la versión 2.x desde arduino.cc. Es el entorno donde escribirás, compilarás y cargarás el código en el Wemos D1 mini Placa de desarrollo basada en el chip ESP8266. Incorpora WiFi integrado, 11 pines GPIO y se programa con Arduino IDE igual que un Arduino normal. .
2
Soporte ESP8266 en el gestor de placas
Ve a Archivo → Preferencias y añade esta URL:
https://arduino.esp8266.com/stable/package_esp8266com_index.json
Después instala el paquete esp8266 by ESP8266 Community desde el gestor de placas.
3
Seleccionar la placa correcta
En Herramientas → Placa, selecciona LOLIN(WeMos) D1 R2 & mini. Conecta la placa por USB y selecciona el puerto COM Puerto de comunicación serie del ordenador. En Windows aparece como COM3, COM4… En Linux y Mac como /dev/ttyUSB0 o similar. correspondiente.

📦Librerías que necesitas

Instálalas desde Herramientas → Gestionar librerías… en Arduino IDE.

Librería Para qué sirve
Adafruit BMP280 Librería oficial de Adafruit para comunicarse con el sensor BMP280 usando I2C o SPI. Proporciona métodos como readTemperature() y readPressure() que el sketch utiliza directamente. Leer temperatura, presión y altitud del BMP280.
Wire Librería incluida por defecto en Arduino IDE. Gestiona la comunicación I2C entre el microcontrolador y los dispositivos conectados al bus. Comunicación I2C (incluida por defecto).
ArduinoJson Librería que permite crear y leer documentos JSON en C++. Se usa para construir el mensaje que se enviará por MQTT. Construir el mensaje JSON con los datos del sensor.
ESP8266WiFi WiFi del ESP8266 (incluida con la placa).
PubSubClient Cliente MQTT (se usará en Fase 4).
💡 Antes de cargar cualquier código, usa el I2C Scanner Programa de diagnóstico que recorre todas las direcciones I2C posibles (0x01–0x7E) y muestra por el monitor serie qué dispositivos responden. El BMP280 suele responder en la dirección 0x76 o 0x77 según cómo esté soldado el pin SDO. para confirmar que el sensor está correctamente conectado. Si el escáner no detecta ningún dispositivo, revisa el cableado SDA/SCL, la alimentación y el GND común.
bmp280_local.ino · Lectura local del sensor (fragmento)
// ── Librerías necesarias ─────────────────────────────────────────────
#include <Wire.h>             // Comunicación I2C
#include <Adafruit_BMP280.h>  // Sensor de temperatura y presión
#include <ArduinoJson.h>      // Construcción del mensaje JSON

// ── Objeto del sensor ─────────────────────────────────────────────────
Adafruit_BMP280 bmp;           // Instancia del BMP280 por I2C

// ── Presión a nivel del mar (para calcular altitud estimada) ──────────
#define SEALEVELPRESSURE_HPA (1013.25)

void setup() {
  Serial.begin(115200);
  Wire.begin();               // Inicia el bus I2C

  // ── I2C Scanner: muestra qué dispositivos hay conectados ──────────────
  for (uint8_t addr = 1; addr < 127; addr++) {
    Wire.beginTransmission(addr);
    if (Wire.endTransmission() == 0)             // 0 = ACK: dispositivo presente
      Serial.printf("[I2C] Encontrado: 0x%02X\n", addr);
  }

  // ── Inicializa el BMP280 (prueba 0x76, luego 0x77) ────────────────────
  if (!bmp.begin(0x76)) {
    if (!bmp.begin(0x77)) {
      Serial.println("*** BMP280 NO DETECTADO. Revisa el cableado ***");
      return;
    }
  }
  Serial.println("BMP280 inicializado correctamente");

  // ── Configura oversampling y filtro para lecturas estables ────────────
  bmp.setSampling(
    Adafruit_BMP280::MODE_NORMAL,    // Modo continuo
    Adafruit_BMP280::SAMPLING_X2,    // Temperatura: oversampling x2
    Adafruit_BMP280::SAMPLING_X16,   // Presión: oversampling x16
    Adafruit_BMP280::FILTER_X16,     // Filtro IIR: suaviza lecturas
    Adafruit_BMP280::STANDBY_MS_500  // Espera 500 ms entre medidas internas
  );
}

void loop() {
  // ── Lee los datos del sensor ──────────────────────────────────────────
  float tempC   = bmp.readTemperature();              // Temperatura en ºC
  float pressHP = bmp.readPressure() / 100.0;        // Presión: Pa → hPa
  float altM    = bmp.readAltitude(SEALEVELPRESSURE_HPA); // Altitud estimada en m

  // ── Imprime los valores en el monitor serie ───────────────────────────
  Serial.printf("Temperatura: %.2f ºC  |  Presión: %.2f hPa  |  Altitud: %.1f m\n",
               tempC, pressHP, altM);

  // ── Construye el JSON para preparar la Fase 4 (MQTT) ─────────────────
  DynamicJsonDocument doc(512);
  doc["sensor"]      = "BMP280";
  doc["temperatura_c"] = tempC;
  doc["presion_hpa"]   = pressHP;
  doc["altitud_m"]     = altM;

  serializeJsonPretty(doc, Serial);   // Muestra el JSON formateado en consola
  Serial.println();

  delay(5000);   // Espera 5 segundos antes de la siguiente lectura
}
